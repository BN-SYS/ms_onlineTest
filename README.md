# 멘사 온라인 테스트 시스템

## [프로토타입 보기](https://bn-sys.github.io/ms_onlineTest/v3)

## 요구사항 요약

기존 멘사코리아 입회테스트 사이트에 회원/비회원 모두 무료로 응시 가능한 3단계 인지능력 테스트 기능을 추가 개발한다.<br>
결과 확인 시 2단계 결제 옵션(기본 결과/상세 리포트)을 제공하며, 단계별 문제 은행 기반 랜덤 출제, 응시자 출생년도 정보를 기반으로 연령 보정 채점, 비정상 응시 패턴 차단, 관리자 문제/로그 관리 기능을 포함한다.<br>
관리자페이지 또한 기존에 만들어져 있는 사이트에 '온라인테스트 관리' 메뉴를 추가 개발한다.

## 1. 프로젝트 목적 및 배경

### 기획 배경

- 기존 입회 테스트 외 별도 온라인 체험형 콘텐츠 신설 (테스트 작명 필요 / IQ 사용 금지)
- 멘사 브랜드 기반 온라인 테스트 서비스 제공
- 무료 응시 → 결과 유료화 수익 모델

### 목표

- 신규 유입 확보 및 브랜드 경험 제공
- 응시 데이터 축적 및 분석 기반 마련
- 2단계 결제 구조로 수익성 확보

## 3. 테스트 구조 상세

### 3-1. 단계별 특징

| 단계 | 문제 수 | 시간 제한 | 측정 영역 | 문제 은행 규모 | 기본점수 | 문제당 배점 |
|------|---------|----------|----------|--------------|---------|-----------|
| 1단계 | 관리자 설정 | 관리자 설정 | 레이븐스 매트릭스 (도형 패턴 인식) | 관리자 설정 N개 | 관리자 설정 | (100 - 기본점수) / 문제수 |
| 2단계 | 관리자 설정 | 관리자 설정 | 수리 연산 능력 | 관리자 설정 N개 | 관리자 설정 | (100 - 기본점수) / 문제수 |
| 3단계 | 관리자 설정 | 관리자 설정 | 기억력 또는 상식 (웩슬러 요소) | 관리자 설정 N개 | 관리자 설정 | (100 - 기본점수) / 문제수 |

#### 설정 예시 (관리자에서 조정 가능)

- 1단계: 15문제, 기본점수 40점, 문제당 4점, 제한시간 30분
- 2단계: 5문제, 기본점수 60점, 문제당 8점, 제한시간 10분
- 3단계: 5문제, 기본점수 60점, 문제당 8점, 제한시간 10분

### 3-2. 문제 출제 로직

**문제 은행 구조**

- 각 단계별로 독립적인 문제 은행(pool) 보유
- 예: 1단계 문제 은행 N개 → 응시 시 랜덤 N개 선택
- 사용자마다 다른 문제 조합 출제
- 🔴 비정상 패턴 감지로 인한 단계 재시작 시에는 새로운 랜덤 문제 세트로 출제 (동일 문제 재출제 방지 권장)

**문제 등록 형식**

- 문제(질문)는 아래 3가지 형식 중 **1가지만** 선택하여 등록
  - 텍스트만
  - 파일첨부(이미지)만
  - 텍스트 + 파일첨부(이미지) 혼합
- 보기(선택지)는 **파일첨부(이미지)만** 가능 (최소 2개 ~ 최대 8개)

**채점 기준**

- 정답 채점은 **문항 ID**와 **보기 ID** 기반으로 수행
- 응시자가 선택한 보기 ID와 정답 보기 ID를 비교하여 정오 판별

**랜덤 처리 규칙**

- **문제 선택**: 문제 은행에서 랜덤 추출 후, 추출된 순서대로 출제 (출제 후 순서 변경 없음)
- **보기 순서**: 매 응시마다 랜덤 셔플 (보기 ID는 고정, 표시 순서만 변경)
- **보기 개수**: 문제마다 2~8개 유동적 (문제 등록 시 설정)

### 3-3. 검증 로직 (비정상 응시 차단)

**각 단계 완료 후 검증 (관리자 설정값 기준)**

- 정답률 N% 미만 → 해당 단계 재응시
- 평균 응답시간 N초 미만 → 해당 단계 재응시
- 미통과 시 메시지: "정답률이 너무 낮거나 비정상적인 응시 패턴이 감지되었습니다. 문제를 충분히 읽고 신중히 응답해주세요."

**재응시 처리 원칙**

- 재응시 시 해당 단계의 문제 세트는 새롭게 랜덤 구성
- 🔴 비정상 응시 감지로 재시작되어도 **하나의 응시 로그**로 저장 (응시자 정보 유지, 재응시 횟수 및 비정상 플래그 누적 기록)

## 5. 점수 산출 로직

### 5-1. 점수 체계 개요

**100점 만점 체계**

- 각 단계별로 독립적인 점수 산출 (최대 100점)
- 종합 점수는 1단계(×3), 2단계(×1), 3단계(×1), 작업속도(×1), 메타인지(×1)의 **가중 평균 (/7)**
- 연령 보정은 단계별 점수에 적용 (각 단계 최대 100점 초과 불가)
- 종합 점수 산출 후 **오프라인 입회테스트 유도를 위한 전체 점수 상향 보정** 적용 (최대 100점 초과 불가)

**등급 체계**

- 결과 화면에 백분위 대신 **등급(하위권 / 중위권 / 상위권 / 최상위권)** 으로 표시
- 등급 기준은 관리자 설정에서 점수 구간별로 조정 가능
- 예시 기준 (확정 전 임시):

| 등급 | 종합점수 구간 | 의미 |
|------|------------|------|
| 하위권 | 0 ~ 59점 | 평균 이하 |
| 중위권 | 60 ~ 79점 | 평균 수준 |
| 상위권 | 80 ~ 89점 | 평균 이상 |
| 최상위권 | 90점 이상 | 멘사 입회 도전 권장 |

> 상향 보정값 적용 후 최종 점수 기준으로 등급 산출

### 5-2. 단계별 점수 계산 공식

```javascript
// 1. 단계별 원점수 계산 (문항ID/보기ID 기반 정답 비교)
단계_원점수 = 기본점수 + (정답개수 × 문제당배점)

// 2. 연령 보정 적용
단계_점수 = 단계_원점수 + 연령보정값
if (단계_점수 > 100) 단계_점수 = 100

// 3. 작업속도 점수 계산
// - 최소소요시간 이하 완료 시 100점
// - 제한시간 내 완료 시 최소 60점 보장 (시간 내 완료 자체를 인정)
// - 최소소요시간 초과 ~ 제한시간 이하: 60~100점 선형 환산
작업속도_점수 = (실제소요시간 <= 최소소요시간)
  ? 100
  : Math.max(60, 100 - ((실제소요시간 - 최소소요시간) / (제한시간 - 최소소요시간)) * 40)
// 핵심: 범위를 100~60점(40점 폭)으로 설정하여 제한시간 내 완료 시 최소 60점 보장

// 4. 메타인지 점수 계산
// - 총 25문제 기준, 문제당 4점 (=100점 만점)
// - 예상 정답수와 실제 정답수의 차이만큼 감점
// - 실제정답수 기준 최대 100점, 오차 1문제당 4점 감점
메타인지_점수 = Math.max(0, 100 - (Math.abs(예상정답수 - 실제정답수) * 4))

// 5. 가중 종합점수 계산 (1단계 가중치 3, 나머지 각 1, 합산 /7)
종합점수_raw = (1단계점수 × 3 + 2단계점수 × 1 + 3단계점수 × 1
              + 작업속도점수 × 1 + 메타인지점수 × 1) / 7

// 6. 전체 점수 상향 보정 (관리자 설정값, 최대 100점 제한)
종합점수_최종 = Math.min(100, 종합점수_raw + 상향보정값)
```

### 5-3. 점수 계산 예시

**가정 (관리자 설정값)**

- 1단계: 15문제, 기본점수 40점, 문제당 4점, 제한시간 30분, 최소소요시간 5분
- 2단계: 5문제, 기본점수 60점, 문제당 8점, 제한시간 10분, 최소소요시간 2분
- 3단계: 5문제, 기본점수 60점, 문제당 8점, 제한시간 10분, 최소소요시간 2분
- 응시자: 35세 (연령 보정 +4점)
- 전체 실제소요시간: 38분 (최소 9분, 제한 50분)
- 예상 정답수: 20개 / 실제 정답수: 19개
- 상향 보정값: +5점 (관리자 설정)

#### 1단계 점수

```
원점수 = 40 + (12 × 4) = 88점
연령보정 후 = 88 + 4 = 92점
```

#### 2단계 점수

```
원점수 = 60 + (4 × 8) = 92점
연령보정 후 = 92 + 4 = 96점
```

#### 3단계 점수

```
원점수 = 60 + (3 × 8) = 84점
연령보정 후 = 84 + 4 = 88점
```

#### 작업속도 점수

```
실제소요시간(38분) > 최소소요시간(9분) → 선형 환산 구간
작업속도_점수 = 100 - ((38 - 9) / (50 - 9)) × 40
             = 100 - (29 / 41) × 40
             = 100 - 28.3
             ≈ 72점
(제한시간 내 완료이므로 최소 60점 보장, 72점 적용)
```

#### 메타인지 점수

```
오차 = |20 - 19| = 1문제
메타인지_점수 = 100 - (1 × 4) = 96점
```

#### 가중 종합점수

```
종합점수_raw = (92 × 3 + 96 × 1 + 88 × 1 + 72 × 1 + 96 × 1) / 7
            = (276 + 96 + 88 + 72 + 96) / 7
            = 628 / 7
            ≈ 89.7점

종합점수_최종 = min(100, 89.7 + 5) = 94.7점 → 95점 (반올림)
```

### 5-4. 연령 보정

**연령대별 보정값 (단계별 점수 산출 시 적용)**

| 연령대 | 보정 점수 | 근거 |
|--------|---------|------|
| ~29세 | +0점 | 기준 연령대 |
| 30~39세 | +4점 | 인지 능력 자연 감소 보정 |
| 40~49세 | +8점 | 동일 |
| 50~59세 | +12점 | 동일 |
| 60세 이상 | +16점 | 동일 |

**보정 적용 예시**

- 25세, 기본 88점 → 88 + 0 = 88점
- 35세, 기본 88점 → 88 + 4 = 92점
- 45세, 기본 88점 → 88 + 8 = 96점
- 55세, 기본 88점 → 88 + 12 = 100점 (제한)
- 65세, 기본 88점 → 88 + 16 = 100점 (제한)

### 5-5. 전체 점수 상향 보정

**목적**

- 오프라인 멘사코리아 입회테스트 신청을 유도하기 위해 종합점수 산출 후 추가 상향 보정 적용
- 관리자 설정 페이지에서 보정값 조정 가능

**적용 규칙**

```javascript
// 종합점수(가중평균) 산출 완료 후 적용
종합점수_최종 = Math.min(100, 종합점수_raw + 상향보정값)
// 상향보정값: 관리자 설정 (예: +5점)
// 최종 점수는 100점을 초과할 수 없음
```

**상향 보정 적용 흐름**

```
단계별 원점수 산출
  → 연령 보정 적용 (단계별, 최대 100점)
  → 가중 종합점수 계산 (작업속도/메타인지 포함)
  → 전체 상향 보정 적용 (최대 100점 제한)
  → 최종 종합점수 확정
```

## 7. 결과 출력

### 7-1. 기본 결과 포함 항목

```
┌────────────────────────────────┐
│        {테스트명} Result        │
├────────────────────────────────┤
│                                │

│  진위확인 코드: A3F2-8D1C      │
├────────────────────────────────┤
│  [상세 리포트 보기] 버튼        │
│  (차액 결제 안내)              │
└────────────────────────────────┘
```

**발급번호 생성 규칙**

```
형식: MS-{년도}-{일련번호 6자리}
예시: MS-2026-000123
일련번호 = 응시 로그 ID (자동 증가)
```

**진위확인 코드 생성 규칙**

```
형식: {4자리 해시}-{4자리 해시}
예시: A3F2-8D1C
생성 방법: MD5(이름 + 이메일 + 발급번호 + 타임스탬프).substring(0, 9).toUpperCase()
```

### 7-3. 표준편차 그래프 구현

**구현 방식: Chart.js (필수)**

- Chart.js 라이브러리를 사용하여 정규분포 곡선을 동적으로 렌더링
- 응시자 점수를 기반으로 실시간 계산 후 위치 표시

**그래프 구성 (4개)**

1. 종합 점수 분포 그래프
2. 1단계 (시각 추론) 점수 분포 그래프
3. 2단계 (논리 사고) 점수 분포 그래프
4. 3단계 (종합 인지) 점수 분포 그래프

### 7-4. 개선 방향 멘트

**점수대별 멘트 예시 (5~10가지 준비)**

```
상위 1% 이내:
"멘사 회원 수준의 뛰어난 인지 능력을 보유하고 계십니다.
공식 멘사 입회 테스트 합격 가능성이 매우 높습니다."

상위 2~5%:
"우수한 인지 능력을 가지고 계십니다.
멘사 공식 테스트에 도전해보시길 권장드립니다."

상위 10~25%:
"평균 이상의 능력을 보유하고 계십니다.
추가 학습과 훈련을 통해 더욱 발전할 수 있습니다."

상위 50%:
"평균 수준의 인지 능력을 가지고 계십니다.
꾸준한 노력으로 향상 가능합니다."

하위 50%:
"아직 발전 가능성이 많습니다.
충분한 휴식 후 재도전을 권장합니다."
```

**정규분포 파라미터 (관리자 설정 또는 고정값)**

| 파라미터 | 기본값 | 설명 |
|---------|------|------|
| 평균 (mean) | 75점 | 전체 응시자 평균 점수 기준 |
| 표준편차 (stdDev) | 12점 | 점수 분산 정도 |

> 평균/표준편차는 실 응시 데이터 누적 후 관리자 페이지에서 실측값으로 교체 가능

## 10. 관리자 기능

### 10-1. 테스트 설정 관리

**단계별 설정 항목 (관리자 페이지 `test-settings.html`)**

| 설정 항목 | 설명 |

### 10-2. 문제 관리

**기능 목록**

- 단계별 문제 등록 / 수정 / 삭제
- 단계별 문제 은행 현황 조회 (등록 수 확인)


**문제 등록 규칙**

- 문제(질문)는 텍스트만 / 이미지만 / 텍스트+이미지 중 **1가지 형식** 선택
- 보기(선택지)는 **이미지 파일 첨부만** 가능 (텍스트 입력 불가)
- 보기 개수는 최소 2개 ~ 최대 8개, 미사용 보기 입력란은 자동 비활성화
- 정답은 라디오 버튼으로 1개만 선택
- 이미지 첨부 시 등록/조회/수정 모든 화면에서 썸네일 노출
- 채점은 문항 ID와 보기 ID 기반으로 수행 (보기 순서가 랜덤 셔플되어도 ID로 정답 식별)

## 11. 문의하기 시스템

### 11-1. 변경 사항

**기존 (v1)**

- 로그인 후 문의 가능
- 문의 내역 DB 저장

**변경 요구사항 (v2)**

- **폼메일 방식**으로 변경
- 로그인 불필요
- 문의 양식 작성 → 관리자 이메일로 자동 전송
- 관리자 페이지에서 수신 이메일 변경 가능

**변경 이유**

- 운영 담당자 변경 또는 외부 위탁 대비
- 간편한 관리 및 유지보수
